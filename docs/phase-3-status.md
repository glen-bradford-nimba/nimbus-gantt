## 7. Resolver workaround notes (for future phases)

`@nimbus-gantt/app`'s `package.json` has three attributes that make Turbopack resolution tricky when consumed via a `file:` link:

1. `main: "src/index.ts"` â€” most bundlers want JS at the main entry. Workaround: alias the bare package name to the explicit `./node_modules/@nimbus-gantt/app/src/index.ts` path via `turbopack.resolveAlias`.
2. `exports: { ".": { ... } }` â€” the restrictive exports field blocks all subpath imports, including `@nimbus-gantt/app/src/templates/cloudnimbus/styles.css`. Workaround: alias a virtual subpath `@nimbus-gantt/app/styles.css` to the real file system path.
3. Symlinked file: link into `../nimbus-gantt/packages/app` lives outside the Turbopack default filesystem root. Workaround: set `turbopack.root: '..'` to widen the scope.

All three aliases live in `next.config.ts` so they stay inside the Phase 3 scope envelope. Phase 6 (docs) may want to recommend upgrading the `@nimbus-gantt/app` package.json to ship a `dist/` build and list subpath exports so downstream React consumers don't need any of this.
